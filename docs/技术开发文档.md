## 1. 系统概述

**项目名称**：黄金ETF技术分析系统（gold_etf_web）  
**项目类型**：前后端分离的 Flask Web 应用  
**核心功能**：股票/ETF 技术指标可视化分析，支持 K线图、MACD、KDJ、RSI、布林带等技术指标计算与展示，提供资金流向分析与新闻资讯。  
**默认标的**：华夏黄金ETF（518880）

### 1.1 技术栈

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| 后端框架 | Flask 3.x | 轻量级 WSGI 框架 |
| 用户认证 | Flask-Login | Session 管理 |
| 数据库 | SQLite | 用户数据 + K线缓存 |
| 数据获取 | requests | HTTP 请求 |
| 数据处理 | pandas | K线数据处理 |
| 技术指标 | numpy + pandas | 自主实现 |
| 前端渲染 | ECharts 5.4 | K线及各类图表 |
| 前端协议 | REST JSON API | 前后端分离 |

## 4. 核心模块详解

### 4.1 技术指标（indicators.py）

所有指标函数接收含 `开盘/最高/最低/收盘/成交量` 列的 DataFrame，返回添加指标列的新 DataFrame（不修改原对象）。

| 函数 | 新增列 | 说明 |
|------|--------|------|
| `_add_ma()` | MA5, MA10, MA20 | 简单移动平均线 |
| `_add_change()` | 涨跌幅, 涨跌额 | 收盘价变动百分比 |
| `_add_macd()` | MACD, MACD_SIGNAL, MACD_HIST | 12/26/9 参数 EMA |
| `_add_kdj()` | K, D, J | 9日 RSV 随机指标 |
| `_add_rsi()` | RSI | 14日相对强弱指数 |
| `_add_bollinger()` | BB_MID, BB_UPPER, BB_LOWER | 20日布林带（2σ） |
| `_add_money_flow()` | 资金净流入, 累计净流入, 主力净流入, 主力占比 | 估算资金流向 |
| `_add_obv()` | OBV | 能量潮指标 |
| `_add_volatility()` | VOLATILITY | 年化波动率（20日窗口） |

---

### 4.2 数据获取与缓存（gold_data.py）

**缓存策略**：

1. **无日期范围请求**：按需刷新 — 数据库有当天数据直接用缓存；无当天数据则仅拉取缺失日期增量补入。
2. **有日期范围请求**：始终从网络获取当天数据（即使在范围内），历史数据走缓存。
3. **去重写入**：`INSERT OR REPLACE`，按 `(symbol, date)` 联合唯一键。
4. **主备数据源**：新浪财经 K线接口（主，240分钟周期）→ 腾讯财经（备）。

**数据源**：

- 新浪：`https://money.finance.sina.com.cn/quotes_service/api/json_v2.php/CN_MarketData.getKLineData?symbol={symbol}&scale=240&ma=no&datalen={datalen}`
- 腾讯：`https://web.ifzq.gtimg.cn/appstock/app/fqkline/get?_var=kline_dayqfq&param={symbol},day,,,{datalen},qfq`

### 4.3 股票代码规范化（data.py）

| 输入格式 | 输出 | 说明 |
|---------|------|------|
| `518880` | `sh518880` | 自动识别为沪市ETF |
| `000300` | `sz000300` | 自动识别为深市指数 |
| `sh518880` | `sh518880` | 保持不变 |
| `sz000300` | `sz000300` | 保持不变 |
| 任意中文 | `sh518880`（默认） | 无法识别返回默认值 |

**代码前缀规则**：
- `6xxxxx` → `sh`（沪市主板/科创板）
- `000xxx / 001xxx / 002xxx / 003xxx` → `sz`（深市）
- `300xxx` → `sz`（创业板）
- `8xxxxx` → `bj`（北交所）

### 4.4 信号生成逻辑（gold_data.py）

系统从 6 个维度生成分析信号：

| 信号类型 | 判断条件 | 状态 |
|---------|---------|------|
| 均线多头 | 收盘 > MA5 > MA10 | ✅ 看涨 |
| 均线空头 | 收盘 < MA5 < MA10 | ⚠️ 看跌 |
| 均线混乱 | 其他情况 | ➡️ 观望 |
| MACD强势 | MACD > 0 且 MACD > Signal | ✅ 看涨 |
| MACD弱势 | MACD < 0 且 MACD < Signal | ⚠️ 看跌 |
| MACD反弹 | MACD_HIST > 0 且 MACD < 0 | 📈 谨慎看涨 |
| MACD调整 | 其他 | ➡️ 观望 |
| KDJ超买 | J > 80 | ⚠️ 谨慎 |
| KDJ超卖 | J < 20 | 📈 关注 |
| KDJ中性 | 其他 | ➡️ 正常 |
| RSI超买 | RSI > 70 | ⚠️ 谨慎 |
| RSI超卖 | RSI < 30 | 📈 关注 |
| RSI偏强 | RSI > 50 | ✅ 偏多 |
| RSI偏弱 | RSI < 50 | ⚠️ 偏空 |
| 资金流入 | 累计净流入 > 0 | ✅ 利好 |
| 资金流出 | 其他 | ⚠️ 谨慎 |

**综合建议规则**：
- 看涨信号 > 看跌信号 + 2 → 📈 短期偏多
- 看跌信号 > 看涨信号 + 2 → 📉 短期偏空
- 其他 → ➡️ 震荡整理

### 4.5 网格交易（grid_trade.py）

`get_grid_signal(latest, grid_count, grid_spread, ma_key, macd_ma_key, macd_hist_window, macd_hist_mean)`

支持**均线锚定**和**MACD锚定**两种基准模式：

#### 均线锚定模式（默认）

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| ma_key | str | MA20 | 基准均线，可选 MA5/MA10/MA20/MA60 |
| grid_spread | float | ATR动态 | 手动指定网格总幅度（否则按ATR动态） |

- **基准价** = 选定均线值（锚定市场平均成本）
- **区间** = 均线 ± 2×ATR（限制 5%~30%）
- **动态spread**：无手动值时自动按 `4×ATR / 基准价` 计算

#### MACD锚定模式（macd_ma_key 参数）

| macd_ma_key | 基准值 | 说明 |
|-------------|--------|------|
| MACD | DIF（EMA12-EMA26差值） | 以DIF线作为网格基准 |
| MACD_SIGNAL | DEA（MACD的9日EMA） | 以DEA线作为网格基准 |

MACD模式下，`macd_hist_mean`（MACD_HIST 的20日均值）用于动态调整网格宽度：

| HIST均值范围 | 网格系数 | 市场状态 |
|-------------|---------|---------|
| 0.000 ~ 0.002 | ×0.80 | 震荡 → 缩小网格降低成本 |
| 0.002 ~ 0.005 | ×1.00 | 正常 |
| 0.005 ~ 0.010 | ×1.20 | 趋势偏强 → 适当扩大 |
| 0.010+ | ×1.50 | 强趋势 → 扩大避免频繁触发 |

#### 返回字段（MACD模式额外字段）

| 字段 | 说明 |
|------|------|
| macd_ma_key | 使用的MACD均值类型 |
| macd_hist_mean | MACD_HIST 的20日均值 |
| macd_hist_current | 当前 MACD_HIST 值 |
| grid_adjusted | 是否因MACD趋势调整了网格参数 |

前端 `grid_signals` 字典现在包含 6 个 key：`MA5` / `MA10` / `MA20` / `MA60` / `MACD` / `MACD_SIGNAL`。

### 4.6 信号服务（signal.py）

`SignalSummary` 类和以下函数：

| 函数 | 说明 |
|------|------|
| `get_signal_summary(symbol, datalen)` | 获取指定标的的信号摘要（latest指标 + signals列表） |
| `format_signal_text(summary)` | 格式化为微信公众号可用的纯文本（带emoji） |
| `get_trading_signal(summary)` | 返回简化交易信号：`买入` / `卖出` / `观望` |

综合打分规则：

| 条件 | 加分 |
|------|------|
| 收盘 > MA5 > MA10 | +1 |
| 收盘 < MA5 < MA10 | -1 |
| MACD_HIST > 0 | +1 |
| MACD_HIST < 0 | -1 |
| RSI > 70 | -1 |
| RSI < 30 | +1 |
| KDJ J > 80 | -1 |
| KDJ J < 20 | +1 |

- 得分 ≥ +2 → 买入
- 得分 ≤ -2 → 卖出
- 其他 → 观望

### 4.7 微信公众号发布（publisher.py）

`publish_signal_article(signals_text, trade_signal, symbol, symbol_name)`

## 5. 前端渲染（main.js）

### 5.1 ECharts 图表

| 图表 | 绑定元素 | 类型 | 数据 |
|------|---------|------|------|
| 主图 | `#mainChart` | Candlestick + Line | K线 + MA5/10/20 + 布林带 |
| MACD | `#macdChart` | Line + Bar | MACD/Signal线 + Histogram柱 |
| KDJ | `#kdjChart` | Line | K/D/J 线 + 20/80 参考线 |
| 成交量 | `#volumeChart` | Bar | 成交量（红绿着色） |
| 资金流向 | `#moneyChart` | Bar + Line | 每日净流入 + 累计净流入 |

### 5.2 K线数据格式

**后端返回**：`[[open, close, low, high], ...]`（固定顺序）  
**ECharts 期望**：`[open, close, low, high]`（蜡烛图格式）

前端映射：`[d[0], d[1], d[2], d[3]]` 即 `[开盘, 收盘, 最低, 最高]`

### 5.3 URL 参数同步

页面加载时从 `?symbol=xxx&start_date=xxx&end_date=xxx` 读取初始参数；搜索后通过 `history.replaceState` 更新 URL（不刷新页面，不跳转）。

    open    REAL NOT NULL,
    high    REAL NOT NULL,

    close   REAL NOT NULL,
    volume  REAL NOT NULL,
